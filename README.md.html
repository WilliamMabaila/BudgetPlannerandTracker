1    # Budget Planner and Tracker
2    
3    ## An Application to Manage Personal Finances
4    
5    What will the application do?:
6    - The Budget Planner and Tracker will help users manage their personal finances. It will allow them to set monthly 
7      budgets, track expenses, and visualize their spending habits.
8    
9    Who will use it?
10   - This application is aimed at individuals who want to gain better control over their personal finances 
11     from students to professionals.
12   
13   Why is the project of interest to me?
14   - Given my background in Mathematics and Economics, I am passionate about numbers and financial planning. 
15     This project allows me to apply my academic skills to a practical everyday problem.
16   
17   ## User Stories
18   - As a user, I want to be able to add expenses/incomes to my budget tracker.
19   - As a user, I want to be able to remove expenses/incomes to my budget tracker.
20   - As a user, I want to be able to get total expenses/incomes.
21   - As a user, I want to be able to get net flow of expenses/incomes.
22   - As a user, I want to be able to view a list of my expenses and incomes.
23   - As a user, I want to be able to set a monthly budget for various categories (like groceries, entertainment, etc.).
24   - As a user, I want to be able to save my financial data and load it later to continue where I left off.
25   - As a user, I want to be able to categorize my expenses when I add them, so I can see how my spending is 
26     distributed across different categories.
27   
28   ## Instructions for Grader
29   - You can generate the first required action related to the user story "adding multiple incomes to a financial tracker" by clicking the button labeled "Add Income". A dialog will appear prompting you to enter the source of the income, the amount, and the frequency (e.g., "monthly", "one-time"). After filling in these details, click the "OK" button in the dialog to add the income to your financial tracker.
30   - You can generate the second required action related to the user story "adding multiple expenses to a financial tracker" by clicking the button labeled "Add Expense". A dialog will pop up asking for the category of the expense, the amount, a brief description, and whether it is recurring. Fill out these fields and click "OK" to add the expense to the tracker.
31   - You can locate the visual components, which are the GIFs that appear immediately after adding an income or expense in the resources folder.
32   - You can save the state of the application by clicking the "Save" button located at the bottom of the application window. This will save the current state of your financial tracker, including all incomes and expenses you've added, to a file. You will see a confirmation message once the data has been successfully saved.
33   - You can reload the state of the application from a file by clicking the "Load" button, also found at the bottom of the application window. This action will load previously saved data into the financial tracker, updating the table to reflect the loaded state. A confirmation message will appear to indicate that the data has been successfully loaded.

49   ## Phase 4: Task 3
50   Considering potential refactoring to improve the design one aspect that stands 
51   out is the relationship between `FinancialTracker` and the `Income` and `Expense` classes. 
52   Currently, the `FinancialTracker` class directly manages multiple `Income` and `Expense` objects which can lead to a 
53   high degree of coupling between these classes and may make the `FinancialTracker` class overly complex.
54   
55   To address this, we could introduce a new class named `TransactionManager` that would be responsible for 
56   managing `Income` and `Expense` objects. This class would act as an intermediary between `FinancialTracker` and 
57   the `Income` and `Expense` classes thereby adhering to the Single Responsibility Principle (SRP) by allowing 
58   `FinancialTracker` to focus on core functionalities without directly managing the transactions. The `TransactionManager` 
59   class would have a one-to-many association with both `Income` and `Expense` and a one-to-one association 
60   with `FinancialTracker`. This refactoring could simplify the `FinancialTracker` class, making the system more modular 
61   and easier to maintain.
62   
63   While this introduces an additional layer of abstraction which could be seen as a tradeoff in terms of increased 
64   complexity, it offers the benefit of a cleaner separation of concerns. This makes the system more flexible and open to 
65   future changes.
